App_Rules.md (애플리케이션 운영 및 개발 규칙)
이 문서는 루틴 관리 애플리케이션의 데이터 구조, 활성화 로직 및 UI 동작 규칙을 정의합니다. 모든 개발 단계에서 이 규칙을 최우선으로 준수해야 합니다.

#1. 전체적용
#1.1.사용자의 화면에 출력되는 모든 문장은 한글종성체크함수(getJosa)를 사용하여 '이/가''을/를''이죠/죠'를 구분하여 문법적으로 정확한 문구가 출력되도록 한다.

#2. 홈화면
##2.1. 홈화면의 최상단에는 '홈아이콘줄'이 배치됩니다.
'홈아이콘줄'은 어떤 화면에서도 최상단을 차지합니다. 
단, 특수효과, 예를 들어 루틴 완료 후 축하 애니메이션, 혹은 루틴 그룹 완료 후 축하 애니메이션에서는 홈아이콘줄이 사라질 수 있습니다. 
홈아이콘줄에는 다음 아이콘(메뉴)가 배치됩니다.
1.홈 아이콘. 누르면 홈 화면(activeTab: 'home')으로 이동합니다.
2.설정 아이콘. 누르면 설정 화면(isSettingsOpen: true)으로 이동합니다.
3.루틴 그룹 추가 아이콘. 누르면 루틴 그룹 추가 화면(activeTab: 'add')으로 이동합니다. 
4.통계 아이콘. 누르면 통계 화면(activeTab: 'stats')을 이동합니다. 
5.음성안내 아이콘. 누르면 음성안내기능(userData.isVoiceEnabled)이 켜집니다. 디폴트는 '꺼짐'.
6.체크체크박스(CheckCheckIcon). 사용자가 클릭하면 성장하는 캐릭터가 배치됩니다. 앱의 다른 기능이나 루틴 기록에 영향을 주지 않습니다. 

##2.2. 홈화면의 '홈아이콘줄' 하단에는 '현재날짜박스(HeaderBox)'가 배치됩니다.
##2.3. 홈화면의 '현재날짜박스' 하단에는 '기상시각박스(WakeUp Check-in section)'가 배치됩니다. 
##2.4. 홈화면의 '기상시각박스' 하단에는 루틴그룹목록(userData.routineChunks)이 배치됩니다. 사용자가 설정한 루틴그룹이 없는 경우 이 곳은 비어있습니다 
##2.4.1. 루틴그룹박스 내의 개별루틴목록(chunk.tasks)은 실행상태와 무관하게 숫자순서대로 정렬한다.

#8. 기상시각
사용자는 설정화면에서 기상시각(targetWakeUpTime)을 설정할 수 있다. 
기상시각 전후에 유예시간(earlyWindowMinutes, lateWindowMinutes)이 있다.
유예시간동안 홈화면의 기상시각박스는 '체크인' 상자가 나온다. 이 시간에 '체크인'(handleCheckIn)을 성공하면 '성공'화면과 함께 통계(StatsView)에 '성공'으로 표시된다. 

#9.시간 사용 바
1. 시간 사용 바(TimeBar)
홈화면 HeaderBox 하단에 표시되는 해당 바는 내부적으로 "시간 사용 바" 혹은 **TimeBar**로 명칭하며, 시스템 데이터인 dailyActivityLog를 기반으로 작동한다. 이 데이터는 사용자의 앱 접속 상태와 타이머 작동 여부를 분 단위(1440분)로 기록한다.
2. 데이터 저장 및 백업 보장
dailyActivityLog는 UserData 구조체에 포함되어 있으며, 브라우저의 localStorage에 저장된다. 
백업 및 내보내기 기능(handleExportData) 시 해당 정보가 포함되어 다른 루틴 기록과 함께 안전하게 백업된다.
3. 리셋 기능의 경우
사용자가 '루틴 기록 삭제' 또는 '루틴 전체 삭제' 기능을 이용하더라도, 시간 사용 로그(dailyActivityLog)는 초기화되지 않는다.
시간 사용 바 기록의 삭제는 사용자의 '사용 시간 기록 삭제'를 통해서만 가능하다. 
4. 통계 화면 내 '사용시간통계' 탭 
StatsView에ㅅ 세 번째 탭인 **'사용 시간 통계'**를 통해 기록을 확인할 수 있다. 
5. 사용 시간 통계 화면 구현 규칙 적용
레이아웃: 각 줄마다 왼쪽에는 날짜(예: 4/27), 중앙에는 24시간 시간 사용 바, 오른쪽에는 당일 총 타이머 가동 합계 시간을 표시합니다.
시간 축: 설정된 리셋 시각에 관계없이 항상 **00:00(자정)**을 시작점으로 하여 24시간 동안의 기록을 그리도록 구현했습니다.
독립성: 루틴을 리셋해도 이 통계 화면의 바와 시간 정보는 영향을 받지 않습니다.
정렬: 가장 최근 날짜의 기록이 최상단에 오도록 내림차순으로 정렬하여 표시합니다.
실시간 반영: '오늘'의 데이터는 현재 가동 중인 타이머를 실시간으로 반영하여 계산됩니다.

#10. 루틴 그룹과 루틴

##10.1. 루틴 계층 구조
루틴 그룹 (Routine Group): 모든 개별 루틴의 상위 단위(RoutineChunk)입니다.
트리거 루틴 (Trigger Task): 각 그룹은 반드시 1개의 시작점(Trigger/tasks[0])을 가집니다.
일반 루틴 (Task): 각 그룹은 1개 이상의 개별 루틴(Task)을 포함합니다.

##10.2. 루틴 그룹 활성화 상태 로직 (Status Logic)


###10.2.1. 요일별 활성화 규칙
그룹 설정: 사용자는 그룹별로 활성화 요일(scheduledDays)을 선택할 수 있습니다. (기본값: 월~일 전체 활성)
모든 요일이 비활성화된 그룹도 설정할 수 있습니다. 
개별 루틴 설정: 그룹 내 특정 루틴만 요일별(scheduledDays)로 비활성화할 수 있습니다.
단, 그룹 자체가 비활성인 날에는 개별 루틴을 강제로 활성화할 수 없습니다.

###10.2.2. 기본 판단 변수

시스템은 아래 3가지 판단 변수를 조합하여 현재 그룹의 최종 상태를 결정합니다.
isScheduledDay: 현재 요일이 설정된 활성 요일에 포함되는가?(isChunkScheduledToday)
isManualRested: 활성 요일이지만, 사용자가 **[오늘은 쉬어가기]**(toggleInactive/inactiveChunks)를 눌렀는가?
isManualForced: 비활성 요일이지만, 사용자가 **[활성화하기]**(toggleInactive/forcedActiveDates)를 눌렀는가?

###10.2.3. 4가지 상태 정의 및 동작(getChunkStatus)

| 상태 (Status) | 논리 구조 (Logic) | 설명 | UI 버튼 표시 |
| :--- | :--- | :--- | :--- |
| **(A) 정상 활성** | `isScheduledDay && !isManualRested` | 스케줄에 따른 정상 진행 | [오늘은 쉬어가기] |
| **(B) 스케줄 휴식** | `isScheduledDay && isManualRested` | 하는 날이지만 수동으로 쉼 | [활성화하기] |
| **(C) 정상 비활성** | `!isScheduledDay && !isManualForced` | 쉬는 날이라서 비활성 상태 | [활성화하기] |
| **(D) 강제 활성** | `!isScheduledDay && isManualForced` | 쉬는 날이지만 수동으로 켬 | [오늘은 쉬어가기] |


###10.2.4. 최종 상태 요약 (Summary for UI/UX)
2.4.1. 최종 활성 (Active): 상태 (A) 또는 (D)일 때.
루틴 목록 상단에 노출하며, 모든 알림과 로직이 정상 작동합니다.
2.4.2. 최종 비활성 (Inactive): 상태 (B) 또는 (C)일 때.
목록 하단으로 이동하거나 흐리게 표시(Gray-out)하며, 실행 대기열에서 제외합니다.

###10.2.5. 데이터 초기화 (Reset)
모든 수동 조작 기록(isManualRested, isManualForced)은 매일 리셋 시각(dailyResetHour)에 초기화됩니다.
새로운 날이 시작되면 앱은 다시 isScheduledDay(기본 요일 설정)를 기준으로 구동됩니다.

###10.2.6. 상태 전환 (Toggle Logic)
[D] ↔ [C] 전환: 강제 활성 상태(D)에서 '오늘은 쉬어가기'를 누르면 정상 비활성 상태(C)로 복구된다(toggleInactive).
[B] ↔ [A] 전환: 스케줄 휴식 상태(B)에서 '활성화하기'를 누르면 다시 정상 활성 상태(A)로 복구된다(toggleInactive).

###10.2.7. 수행 데이터의 일시 보존 및 소멸
데이터 보존: 상태 (A) 또는 (D) 상태에서 일부 루틴을 수행한 후 (B) 또는 (C)로 전환하더라도, 이미 기록된 개별 루틴의 수행기록과 시간정보는 삭제되지 않고 메모리에 유지된다.
데이터 복구: 동일한 날에 다시 (A) 또는 (D) 상태로 복귀하면, 유지되던 기존 수행 기록이 UI에 즉시 되살아난다.
최종 소멸: (B) 상태인 채로 리셋 시각을 지나 하루가 종료되면, 보존되던 모든 기록은 삭제되며 해당 루틴 그룹은 최종적으로 '미실행(실패)' 처리된다.
최종 소멸: (C) 상태인 채로 리셋 시각을 지나 하루가 종료되면, 보존되던 모든 기록은 삭제되며 해당 루틴 그룹은 최종적으로 '비활성' 처리된다.

###10.2.8. 통계 및 UI 반영 규칙
할 일 개수(Count): (B)와 (C) 상태(비활성)인 그룹은 홈 화면의 Header Box(HeaderBox) 컴포넌트 '오늘 해야 할 루틴 전체 개수(m)' 합계에서 제외한다.
성공/실패 판정: 상태 (A)(정상 활성)과 (D)(강제 활성)은 최종 통계(StatsView)시 산출시 동일하게 취급된다.
성공/실패 판정: 상태 (B)(스케줄 휴식)는 최종 통계 산출 시 '미실행(실패)'(TaskStatus.FAILED)으로 간주한다.
성공/실패 판정: 상태 (C)(정상 비활성)은 최종 통계 산출 시 계산 대상이 아니다.

#20. 실행화면

##20.1. 화면 구성
사용자가 홈화면의 루틴그룹을 선택하면 해당 루틴그룹의 실행화면(ExecutionView)으로 전환된다. 
실행화면은 루틴그룹제목표시박스(RoutineGroupTitleBox), 현재루틴박스(CurrentTaskBox), 루틴목록박스(TaskList)로 구성된다.
모든 루틴그룹은 1개 이하의 현재 루틴(activeTask)을 가진다. 
현재루틴이 없는 경우 현재루틴박스는 표시되지 않는다. 
현재루틴이 아닌 루틴들은 루틴목록박스만 표시된다. 

## 20.2. 첫번째 루틴
미실행 루틴그룹은 첫번째 루틴(트리거 루틴triggerTask)이 가장 먼저 현재 루틴이 된다. 
루틴 그룹의 첫번째 루틴(트리거 루틴triggerTask)은 '일시정지'(togglePauseTask)'완료'(toggleTask)'완벽'(toggleTask)은 할 수 있지만 '나중에'(laterTask)와 '스킵'(skipTask)은 할 수 없다. 


##20.3. 현재 루틴(Current Task) 선정 우선순위
###20.3.1. 사용자가 현재 루틴에 대해 '나중에'(TaskStatus.LATER), '스킵'(TaskStatus.SKIP), '완료'(TaskStatus.COMPLETED) '완벽'(TaskStatus.PERFECT) 액션을 수행할 때, 해당 루틴은 '현재 루틴' 지위를 즉시 상실한다.
###20.3.2. 새로운 '현재 루틴'을 선정할 때, 방금 액션을 수행한 루틴은 제외하고 우선순위에 따라 다음 후보를 찾는다. 우선순위는 다음과 같다 
-1순위. **미실행 루틴**(TaskStatus.NOT_STARTED) 중 가장 번호가 빠른(순서가 앞선) 루틴.
-2순위. 미실행 루틴이 없을 경우, **'나중에'**(TaskStatus.LATER) 상태인 루틴 중 가장 번호가 빠른 루틴.
-3순위. '나중에' 루틴도 없을 경우, **'일시정지'**(isPaused) 상태인 루틴 중 가장 번호가 빠른 루틴.

###20.3.3.'완료' '완벽' '스킵' 상태의 루틴은 자동적으로는 현재 루틴이 되지 않는다. 다만 사용자의 '이어하기'(handleResumeTask) '다시하기'(onRestart) '시작하기'(handleStartTask) 선택을 통해 현재루틴이 될 수 있다.
###20.3.4.현재 루틴이 타이머 진행중이든 혹은 일시정지중이든 관계없이, 사용자가 목록에 있는 루틴의 '이어하기'(togglePauseTask) '다시하기'(onRestart) '시작하기'(togglePauseTask/handleStartTask) 버튼을 누르면 현재루틴은 자동으로 '일시정지' 상태로 루틴목록으로 내려가고 사용자가 선택한 루틴이 '현재루틴'이 된다.  

###20.3.5. 다음 루틴이 될 루틴이 없으면 현재루틴박스 없이 루틴목록박스만 표시된다. 

##20.4. 다음 루틴 자동 시작 (Next Routine Auto Start)
사용자가 옵션에서 '다음 루틴 자동 시작'(userData.nextRoutineAutoStart)을 설정한 경우, 현재 루틴이 변경되면 변경 후 루틴의 타이머가 즉시 자동으로 시작된다.



##20.6. 일시정지(Pause) 동작
**사용자가 현재 루틴을 '일시정지'(togglePauseTask)하면, 해당 루틴은 계속 현재 루틴으로 남아 있으면서 타이머만 정지(task.isPaused)한다. 현재루틴이 목록으로 내려가거나 다음 루틴으로 자동 전환되지 않는다.**

##20.10.루틴 그룹 완료
###20.10.1. 사용자가 하나의 루틴그룹 내의 활성화된 모든 개별 루틴을 완료/완벽/스킵한 경우, 루틴 그룹 완료라고 한다. 
###20.10.2. 루틴 그룹 완료 직후 축하애니메이션(confetti/CelebrationModal)이 나온다.
###20.10.3. 이미 완료한 루틴 그룹의 개별 루틴도 '이어하기' '다시하기' '시작하기'를 통해 수행할 수 있다. 이 경우에는 루틴 그룹 완료 축하 애니메이션이 나오지 않는다. 그러나 변경된 수행/시간정보는 저장되고 반영된다. 
###20.10.4. 이미 완료한 루틴 그룹를 리셋하여 다시 루틴 그룹 완료를 달성하면 루틴 그룹 완료 축하 애니메이션이 나온다. 
----------------------------------------------------
#25.개별 루틴(Task)의 상태
#25.1. 개별 루틴은 활성화 상태(오늘이 실행요일인 경우), 비활성화 상태(오늘이 실행요일이 아닌 경우), 둘로 나뉜다.
#25.2. 비활성화된 루틴도 '시작하기' 버튼을 통해 활성화할 수 있다. 이 경우 활성화 루틴으로 변경된다.
#25.3. 오늘이 활성화 요일인 개별 루틴, 혹은 한 번 활성화한 개별 루틴을 비활성화하는 것은 불가능하다. 
#25.4. 활성화 루틴은 다시 미실행(TaskStatus.NOT_STARTED), 실행중(TaskStatus.IN_PROGRESS), 일시정지(isPaused), 스킵(TaskStatus.SKIP), 완료(TaskStatus.COMPLETED), 완벽(TaskStatus.PERFECT)으로 나누어진다. 
1. 미실행(TaskStatus.NOT_STARTED): 오늘 루틴에 시작시간(startTime)과 타이머 기록 없음. '시작하기' 가능. 
2. 실행중(TaskStatus.IN_PROGRESS): 해당 루틴 그룹에서 현재루틴. 타이머가 돌아가고 있다. 
3. 일시정지(isPaused): 시작 시간이 있고 저장된 타이머가 있다. 타이머가 돌아가고 있지 않다. '이어하기' '다시하기' 가능.
4. 스킵(TaskStatus.SKIP): 타이머를 저장하지 않는다. 수행에 성공하고 종료된 것으로 본다. '시작하기' 가능.
5. 완료(TaskStatus.COMPLETED): 시간제한루틴의 경우 제한시간을 초과해서, 시간축적루틴의 경우 목표시간을 채우지 못하고 종료한 루틴. '이어하기' '다시하기' 가능. 
6. 완벽(TaskStatus.PERFECT): 시간무관루틴의 경우 시간과 무관하게, 시간제한루틴의 경우 제한시간 안에, 시간축적루틴의 경우 목표시간을 채워서 종료한 루틴. '이어하기' '다시하기' 가능. 


#26. 개별 루틴의 기록과 평균
#26.5. 하루 리셋 시간이 되었을 때, 비활성화된 루틴은 기록되지 않고, (1. 미실행)(2. 실행중)(3. 일시정지) 상태의 루틴은 '실패'(TaskStatus.FAILED)로 기록되고, (4. 스킵)(5. 완료)(6. 완벽) 상태의 루틴은 '성공'(TaskStatus.COMPLETED/PERFECT)으로 기록된다.
#26.6. 평균소요시간(averageDuration)은 (5. 완료)(6완벽) 상태의 루틴에 실제로 소요된 시간(타이머에 기록된 시간)만을 계산한다. (4. 스킵)는 계산하지 않는다.

#28. 완벽한 하루
#28.1. 오늘이 수행일인 모든 루틴그룹을 완료한 경우를, '완벽한 하루'(allGroupsCompleted)라고 한다. 오늘이 수행일인 모든 루틴 그룹을 완료해야하므로 오늘이 쉬는 요일이 아님에도 완료하지 않고 '오늘은 쉬어가기'를 선택한 루틴 그룹이 있다면 완벽한하루가 될 수 없다.  
#28.2. 사용자가 마지막 루틴 그룹을 완료하고 홈 화면으로 돌아오면 '완벽한 하루 축하 애니메이션'(PerfectDayAnimation)이 나온다. 
- **제한**: "완벽한 하루 애니메이션"은 **하루에 단 한 번만** 나타난다. 사용자가 루틴을 리셋하거나, 설정을 변경하더라도 해당 날짜(기상 리셋 시간 기준)에 이미 완료 화면을 보았다면 다시 노출되지 않는다.
- **예외 처리**": 단, 사용자가 설정화면의 '루틴 기록 삭제' 또는 '루틴 전체 삭제'를 통해 기록을 리셋한 후에 새로이 모든 루틴 그룹을 완료한 경우에는 '완벽한 하루 축하 애니메이션'이 나온다.

## 31. 데이터 및 환경 설정

### 31.1. 요일 및 날짜 처리
- **유효성**: 루틴은 사용자가 설정한 '실행 요일'에만 활성화됩니다. 쉬는 날인 루틴은 합산 시간이나 목록에서 기본적으로 제외되거나 비활성화 상태로 표시됩니다.
- **날짜 리셋**: 원칙적으로 사용자가 설정한 '기상 시간' 또는 '리셋 시간'을 기준으로 하루의 시작과 끝을 계산합니다. 단 통계화면의 '사용시간통계"의 시간 사용 바의 경우는 0시를 시작점으로 24시간을 표현한다.
