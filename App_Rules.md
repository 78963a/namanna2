#1. 홈화면
##1.1. 홈화면의 최상단에는 '홈아이콘줄'이 배치됩니다.
'홈아이콘줄'은 어떤 화면에서도 최상단을 차지합니다. 단, 루틴 완료 후 축하 애니메이션, 혹은 루틴 그룹 완료 후 축하 애니메이션에서는 홈아이콘줄이 사라질 수 있습니다. 
홈아이콘줄에는 다음 아이콘(메뉴)가 배치됩니다.
1.홈 아이콘. 누르면 홈 화면으로 이동합니다.
2.설정 아이콘. 누르면 설정 화면으로 이동합니다.
3.루틴 그룹 추가 아이콘. 누르면 루틴 그룹 추가 화면으로 이동합니다. 
4.통계 아이콘. 누르면 통계 화면을 이동합니다. 
5.체크체크박스. 사용자가 클릭하면 성장하는 캐릭터가 배치됩니다. 앱의 다른 기능이나 루틴 기록에 영향을 주지 않습니다. 

##1.2. 홈화면의 '홈아이콘줄' 하단에는 '현재날짜박스'가 배치됩니다.
##1.3. 홈화면의 '현재날짜박스' 하단에는 '기상시각박스'가 배치됩니다. 
##1.4. 홈화면의 '기상시각박스' 하단에는 루틴그룹목록이 배치됩니다. 사용자가 설정한 루틴그룹이 없는 경우 이 곳은 비어있습니다

#10. 루틴 그룹과 루틴

##10.1. 루틴 계층 구조
루틴 그룹 (Routine Group): 모든 개별 루틴의 상위 단위입니다.
트리거 루틴 (Trigger Task): 각 그룹은 반드시 1개의 시작점(Trigger)을 가집니다.
일반 루틴 (Task): 각 그룹은 1개 이상의 개별 루틴을 포함합니다.

##10.2. 루틴 그룹 활성화 상태 로직 (Status Logic)


###10.2.1. 요일별 활성화 규칙
그룹 설정: 사용자는 그룹별로 활성화 요일(scheduledDays)을 선택합니다. (기본값: 월~일 전체 활성)
개별 루틴 설정: 그룹 내 특정 루틴만 요일별로 비활성화할 수 있습니다.
단, 그룹 자체가 비활성인 날에는 개별 루틴을 강제로 활성화할 수 없습니다.

###10.2.2. 기본 판단 변수

시스템은 아래 3가지 판단 변수를 조합하여 현재 그룹의 최종 상태를 결정합니다.
isScheduledDay: 현재 요일이 설정된 활성 요일에 포함되는가?
isManualRested: 활성 요일이지만, 사용자가 **[오늘은 쉬어가기]**를 눌렀는가?
isManualForced: 비활성 요일이지만, 사용자가 **[활성화하기]**를 눌렀는가?

###10.2.3. 4가지 상태 정의 및 동작

| 상태 (Status) | 논리 구조 (Logic) | 설명 | UI 버튼 표시 |
| :--- | :--- | :--- | :--- |
| **(A) 정상 활성** | `isScheduledDay && !isManualRested` | 스케줄에 따른 정상 진행 | [오늘은 쉬어가기] |
| **(B) 스케줄 휴식** | `isScheduledDay && isManualRested` | 하는 날이지만 수동으로 쉼 | [활성화하기] |
| **(C) 정상 비활성** | `!isScheduledDay && !isManualForced` | 쉬는 날이라서 비활성 상태 | [활성화하기] |
| **(D) 강제 활성** | `!isScheduledDay && isManualForced` | 쉬는 날이지만 수동으로 켬 | [오늘은 쉬어가기] |


###10.2.4. 최종 상태 요약 (Summary for UI/UX)
2.4.1. 최종 활성 (Active): 상태 (A) 또는 (D)일 때.
루틴 목록 상단에 노출하며, 모든 알림과 로직이 정상 작동합니다.
2.4.2. 최종 비활성 (Inactive): 상태 (B) 또는 (C)일 때.
목록 하단으로 이동하거나 흐리게 표시(Gray-out)하며, 실행 대기열에서 제외합니다.

###10.2.5. 데이터 초기화 (Reset)
모든 수동 조작 기록(isManualRested, isManualForced)은 매일 리셋 시각에 초기화됩니다.
새로운 날이 시작되면 앱은 다시 isScheduledDay(기본 요일 설정)를 기준으로 구동됩니다.

###10.2.6. 상태 전환 (Toggle Logic)
[D] ↔ [C] 전환: 강제 활성 상태(D)에서 '오늘은 쉬어가기'를 누르면 정상 비활성 상태(C)로 즉시 복구된다.
[B] ↔ [A] 전환: 스케줄 휴식 상태(B)에서 '활성화하기'를 누르면 다시 정상 활성 상태(A)로 복구된다.

###10.2.7. 수행 데이터의 일시 보존 및 소멸
데이터 보존: 상태 (A)에서 일부 루틴을 수행한 후 (B)로 전환하더라도, 이미 기록된 개별 루틴의 수행기록과 시간정보는 삭제되지 않고 메모리에 유지된다.
데이터 복구: 동일한 날에 다시 (A) 상태로 복귀하면, 유지되던 기존 수행 기록이 UI에 즉시 되살아난다.
최종 소멸: (B) 상태인 채로 리셋 시각을 지나 하루가 종료되면, 보존되던 모든 기록은 삭제되며 해당 루틴 그룹은 최종적으로 '미실행(실패)' 처리된다.

###10.2.8. 통계 및 UI 반영 규칙
할 일 개수(Count): (B)와 (C) 상태(비활성)인 그룹은 홈 화면의 '오늘 해야 할 루틴 개수' 합계에서 제외한다.
성공/실패 판정: 상태 (B)(스케줄 휴식)는 최종 통계 산출 시 '미실행(실패)'으로 간주한다.

#20. 실행화면

#20.1. 화면 구성
사용자가 홈화면의 루틴그룹을 선택하면 해당 루틴그룹의 실행화면으로 전환된다. 
실행화면은 루틴그룹제목표시박스, 현재루틴박스, 루틴목록박스로 구성된다.
모든 루틴그룹은 1개 이하의 현재 루틴을 가진다. 
현재루틴이 없는 경우 현재루틴박스는 표시되지 않는다. 
현재루틴이 아닌 루틴들은 루틴목록박스에 표시된다. 

#20.3. 현재 루틴(Current Task) 선정 우선순위
아래의 순서로 현재 루틴을 결정합니다:
1. **미실행 루틴** 중 가장 번호가 빠른(순서가 앞선) 루틴.
2. 미실행 루틴이 없을 경우, **'나중에'** 상태인 루틴 중 가장 번호가 빠른 루틴.
3. '나중에' 루틴도 없을 경우, **'일시정지'** 상태인 루틴 중 가장 번호가 빠른 루틴.


#20.4. 다음 루틴 자동 시작 (Next Routine Auto Start)
- 사용자가 옵션에서 '다음 루틴 자동 시작'을 설정한 경우, 현재 루틴이 나중에/스킵/완료/완벅이 되면 위 우선순위에 따라 선정된 다음 루틴이 즉시 자동으로 시작되어야 합니다.


#20.5. 트리거 루틴
루틴 그룹의 첫번째 루틴(트리거 루틴triggerTask)은 '나중에'와 '스킵'을 할 수 없다. 

#20.6. 일시정지(Pause) 동작
- **상태 유지**: 사용자가 현재 루틴을 '일시정지'하더라도, 해당 루틴은 계속 **현재 루틴**으로 남아야 합니다. 목록으로 내려가거나 다음 루틴으로 자동 전환되지 않습니다.

## 11. 핵심 동작 로직 (Core Logic)

### 11.1. "완벽한 하루" 노출 규칙
- **제한**: "완벽한 하루" 화면(모든 루틴 완료 시 노출)은 **하루에 단 한 번만** 나타나야 합니다.
- **예외 처리**: 사용자가 루틴을 리셋하거나, 설정을 변경하더라도 해당 날짜(기상 리셋 시간 기준)에 이미 완료 화면을 보았다면 다시 노출되지 않습니다.





### 21.2. 디자인 수정 및 주석
- 스타일(글자 크기, 색상 등) 관련 코드에는 사용자가 직접 코드를 수정할 수 있도록 안내 주석이 포함되어야 합니다.
- **스타일 관리**: `phrases.json` 파일의 `execution_settings` 및 각 스타일 섹션을 통해 글자 크기와 색상을 중앙 관리합니다.

## 31. 데이터 및 환경 설정

### 31.1. 요일 및 날짜 처리
- **유효성**: 루틴은 사용자가 설정한 '실행 요일'에만 활성화됩니다. 쉬는 날인 루틴은 합산 시간이나 목록에서 기본적으로 제외되거나 비활성화 상태로 표시됩니다.
- **날짜 리셋**: 사용자가 설정한 '기상 시간' 또는 '리셋 시간'을 기준으로 하루의 시작과 끝을 계산합니다.

## 41. 개발 보안 및 유지보수 규칙
- 기존에 구현된 기능이나 디자인의 **의도된 동작**이 변경되지 않도록 수정 시 항상 전체 로직을 점검합니다.
- 새로운 기능을 추가할 때도 위에서 정의된 **현재 루틴 우선순위**와 **일시정지 상태 유지** 규칙이 깨지지 않도록 주의합니다.
